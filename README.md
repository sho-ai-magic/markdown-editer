# Markdownエディタ

軽量でシンプルな、ブラウザで動くMarkdown（`.md`）エディタです。ビルドステップも専用アプリのインストールも不要で、PWA（Progressive Web App）としてChromebook・Windows・Mac・Linuxのどこでも同じように動きます。

**🔗 デモ / 公開版: https://sho-ai-magic.github.io/markdown-editer/**

編集画面に書いたMarkdownがその場でプレビューされ、書式ツールバーのボタンで記法を覚えなくても太字・見出し・リストなどを挿入できます。生成AIとMarkdownをやり取りする機会が増えたことを踏まえ、文字数・概算トークン数カウンターや、Googleドキュメント等にそのまま貼り付けられる書式付きコピー機能も備えています。

## 特徴

- **編集 + リアルタイムプレビュー**の2ペイン表示（テーブル記法対応）
- **書式ツールバー**: 太字・斜体・取り消し線・見出し・箇条書き・番号付きリスト・タスクリスト・引用・コード・表・リンクをボタン一つで挿入
- 見出し・リスト・引用・コードの**簡易シンタックスハイライト**（色はカスタマイズ可能）
- **文字数・概算トークン数カウンター**: 生成AIのチャットに貼り付ける前に分量を把握できる（完全ローカル計算、外部通信なし）
- **Markdownソース／書式付きコピー**: ワンクリックでMarkdownソースをコピー。「書式付き」コピーならGoogleドキュメントやWordに貼り付けても見出し・太字・表がそのまま反映される
- **目次サイドバー**: 見出し一覧をクリックで編集画面・プレビューの両方がジャンプ
- **ダークモード/ライトモード**切り替え（設定は再起動後も保持）
- ファイルの**「開く」「上書き保存」「名前を付けて保存」**（Ctrl+O / Ctrl+S / Ctrl+Shift+S）、ウィンドウへの**ドラッグ&ドロップ**でも開ける
- `.md` ファイルの**関連付け起動**（ダブルクリックでこのアプリが開く。インストール済みPWAのみ）
- **モバイル対応**: スマートフォンでは目次がハンバーガーメニューのドロワーになり、編集/プレビューはタブで切り替え表示
- **オフライン動作**: PWAとしてインストールすればネットワークなしでも起動・利用できる

## 使い方

### 1. アプリとしてインストールする（推奨）

1. Chrome/Edgeで上記のデモURLを開く
2. アドレスバー右端の「インストール」アイコンをクリック
3. 「インストール」を選ぶと、デスクトップやシェルフ（Chromebookの場合）にアイコンが追加される

インストール後はオフラインでも起動できます。

### 2. `.md` ファイルをダブルクリックで開けるようにする

1. 上記の手順でアプリをインストールしておく
2. OSのファイルアプリで `.md` ファイルを右クリック →「アプリで開く」→「その他のアプリ」からこのアプリを選び、「常に使用する」にチェック

以降、`.md` ファイルをダブルクリックするとこのアプリで開きます（Chromebookの「ファイル」アプリ、Windows/Macの「プログラムから開く」いずれも対応）。

### 3. 基本操作

| 操作 | 方法 |
|---|---|
| ファイルを開く | ツールバーの「開く」/ Ctrl+O / ウィンドウへドラッグ&ドロップ |
| 上書き保存 | 「保存」/ Ctrl+S |
| 名前を付けて保存 | 「名前を付けて保存」/ Ctrl+Shift+S |
| 太字 / 斜体 | 編集画面上部の **B** / *I* ボタン、または Ctrl+B / Ctrl+I |
| 見出し | **H** ボタン（クリックのたびにH1→H2→H3→解除と切り替わる） |
| 箇条書き・番号付き・タスクリスト・引用 | 各ボタン（選択した行に適用。もう一度押すと解除） |
| コード | コードボタン（複数行選択でコードブロック、単一行はインラインコード） |
| 表・リンク | 各ボタン（リンクはURL部分がすぐ入力できるよう選択された状態で挿入） |
| Markdownソースをコピー | 「コピー」ボタン |
| 書式付きでコピー | 「書式付き」ボタン（Googleドキュメント等に貼ると見出しや太字が反映される） |
| 目次の表示/非表示 | 「目次」ボタン（ハンバーガーメニュー） |
| 色のカスタマイズ | 「設定」ボタン（「既定の色に戻す」で一括リセット） |
| テーマ切り替え | 太陽/月アイコンのボタン |

未保存の変更があるときはファイル名の横に「●」が表示され、そのまま閉じようとすると確認ダイアログが出ます。

書式ツールバーの「◯◯文字 / 約◯◯トークン」は、生成AIのチャットに貼り付ける際の分量の目安です。厳密なトークナイザーは使わず、日本語などの全角文字は1文字≒1トークン、半角英数字は4文字≒1トークンという簡易な概算のため、実際の値とは多少ずれます。

## 対応ブラウザ

| ブラウザ | 対応状況 |
|---|---|
| Chrome / Edge（ChromeOS・Windows・Mac・Linux） | 全機能対応 |
| Firefox / Safari | 編集・プレビューは利用可。保存は「ダウンロード形式」になり、ファイル関連付け起動は不可 |

## 自分でホストする（フォーク・セルフデプロイ）

このリポジトリをフォークすれば、自分のGitHub Pagesで同じアプリを公開できます。

1. リポジトリをフォークする
2. フォーク先のリポジトリで **Settings → Pages** を開く
3. 「Build and deployment」の **Source** を **GitHub Actions** に設定する
4. `main` ブランチへのpushで `.github/workflows/pages.yml` が自動的にデプロイし、`https://<あなたのユーザー名>.github.io/<リポジトリ名>/` で公開される

ローカルで動作確認する場合はビルド不要で、リポジトリ直下で以下を実行するだけです。

```sh
python3 -m http.server 8000
# http://localhost:8000 を開く
```

## 開発・技術構成

- ビルドステップなしの静的Webアプリ（Vanilla JS）。`index.html` をHTTPで配信すればそのまま動く
- エディタ部品: [CodeMirror 5](https://codemirror.net/5/)（MITライセンス、`vendor/` に同梱）。日本語IMEとの相性が良く、ビルド不要で組み込める
- Markdown変換: [markdown-it](https://github.com/markdown-it/markdown-it)（MITライセンス、`vendor/` に同梱）
- オフライン対応: Service Worker（`sw.js`）。**アプリのファイルを更新したら `sw.js` の `CACHE_VERSION` を上げること**
- ファイルの読み書きにはFile System Access APIを使用（Chrome/Edge系ブラウザで上書き保存に対応。未対応ブラウザはダウンロード形式にフォールバック）
