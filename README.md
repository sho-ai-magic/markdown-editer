# Markdownエディタ（PWA版）

軽量でシンプルなMarkdown（`.md`）エディタです。ブラウザで動くPWA（Webアプリ）として実装されており、**Chromebookをはじめ、Chrome/EdgeのあるWindows・Mac・Linuxでそのまま使えます**。インストールすれば通常のアプリと同じようにアイコンから起動でき、オフラインでも動作します。

## 主な機能

- **編集 + リアルタイムプレビュー**の2ペイン表示（テーブル記法対応）
- 見出し・リスト・引用・コードの**簡易シンタックスハイライト**（色はカスタマイズ可能）
- **目次サイドバー**: 見出し一覧をクリックで編集画面・プレビューの両方がジャンプ
- **ダークモード/ライトモード**切り替え（設定は再起動後も保持）
- ファイルの**「開く」「上書き保存」「名前を付けて保存」**（Ctrl+O / Ctrl+S / Ctrl+Shift+S）
- ウィンドウへの**ドラッグ&ドロップ**でファイルを開く
- `.md` ファイルの**関連付け起動**（ダブルクリックでこのアプリが開く。インストール済みPWAのみ）
- 編集画面をスクロールするとプレビューが**自動で追従**

## 公開手順（リポジトリ管理者向け・初回のみ）

このリポジトリをGitHub Pagesで公開すると、誰でもURLからアプリを使えるようになります。

1. このブランチを `main` ブランチにマージする
2. GitHubのリポジトリページで **Settings → Pages** を開く
3. 「Build and deployment」の **Source** を **GitHub Actions** に設定する
4. `main` へのpushで自動的にデプロイされ、以下のURLで公開される

   **https://sho-ai-magic.github.io/markdown-editer/**

## 使い方（利用者向け）

### 1. アプリとしてインストールする（推奨）

1. ChromeでアプリのURLを開く
2. アドレスバー右端の「インストール」アイコン（画面のような形のアイコン）をクリック
3. 「インストール」を選ぶと、シェルフ（Chromebookの場合）やデスクトップにアイコンが追加される

インストール後はオフラインでも起動できます。

### 2. `.md` ファイルをダブルクリックで開けるようにする（Chromebook）

1. 上記の手順でアプリをインストールしておく
2. 「ファイル」アプリで `.md` ファイルを右クリック →「アプリで開く」→「その他のアプリ」
3. 「Markdownエディタ」を選び、「常に使用する」にチェック

以降、`.md` ファイルをダブルクリックするとこのアプリで開きます。
（WindowsやMacでも、インストール済みPWAは「プログラムから開く」の候補に表示されます）

### 3. 基本操作

| 操作 | 方法 |
|---|---|
| ファイルを開く | ツールバーの「開く」/ Ctrl+O / ウィンドウへドラッグ&ドロップ |
| 上書き保存 | 「保存」/ Ctrl+S |
| 名前を付けて保存 | 「名前を付けて保存」/ Ctrl+Shift+S |
| 目次の表示/非表示 | 「☰ 目次」ボタン |
| 色のカスタマイズ | 「⚙ 設定」ボタン（「既定の色に戻す」で一括リセット） |
| テーマ切り替え | ☀️/🌙 ボタン |

未保存の変更があるときはファイル名の横に「●」が表示され、そのまま閉じようとすると確認ダイアログが出ます。

## 対応ブラウザ

| ブラウザ | 対応状況 |
|---|---|
| Chrome / Edge（ChromeOS・Windows・Mac・Linux） | 全機能対応 |
| Firefox / Safari | 編集・プレビューは利用可。保存は「ダウンロード形式」になり、ファイル関連付け起動は不可 |

## 開発メモ

- ビルドステップなしの静的Webアプリ（Vanilla JS）。`index.html` をHTTPで配信すればそのまま動く
  - ローカル確認: リポジトリ直下で `python3 -m http.server 8000` → http://localhost:8000
- エディタ部品: [CodeMirror 5](https://codemirror.net/5/)（`vendor/` に同梱）
- Markdown変換: [markdown-it](https://github.com/markdown-it/markdown-it)（`vendor/` に同梱）
- オフライン対応: Service Worker（`sw.js`）。**アプリのファイルを更新したら `sw.js` の `CACHE_VERSION` を上げること**
- 元仕様: Windows版Markdownエディタの機能仕様書（SPEC.md）に基づくPWA実装
